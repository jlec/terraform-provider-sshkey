# Coding style guides

## Generic

- line length: 128 characters

## Python

- Indent with 4 spaces
- Quote strings with double quotes
- Use latest syntax supported by the Python version in use
- Use `is` for comparison to `None`
- Use `isinstance` for type checking
- Avoid using mutable default arguments
- Use `with` for file operations
- Use `enumerate` instead of `range(len())`
- Prefer using f-strings for string formatting
- Use `pathlib` for file paths
- Use type hints for function signatures
- Avoid using single-letter variable names except for counters
- Use `snake_case` for variable and function names
- Use `lowercase_with_underscores` for module names
- Prefer httpx over requests for HTTP requests
- Use `pydantic` for data validation and settings management
- Use `uv` for dependency management

## Yaml

- Indent with 2 spaces
- Use true & false for booleans
- Sort lists alphabetically
